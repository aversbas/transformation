XSLT Trainee Test Problem

Introduction

The document contains a test problem for applicants, a short introduction into DocBook and many links to resources where you can learn more about technologies you are supposed to work with: XML, XSLT, HTML and so on.

The test problem contains a few levels. We assume that a few days should be enough to learn all required technologies and develop a solution for the test problem. In case you complete this work faster then you can implement solutions for optional and bonus tasks. The more levels you complete, the better your chances are.

Problem description

The final goal of the test problem is to create an XSLT transformation which converts an XML document into HTML document. With the test problem description you get the source of this document in the XML format, or strictly speaking in DocBook, which is semantic markup language for technical documentation.

Chapters below contain a brief description of several DocBook elements (or group of elements) and a few examples, thus you can easily use this document to test your transformation.

DocBook elements catalog

This chapter contains descriptions of the most commonly used DocBook elements with samples. We don't consider structural elements likebook,articleorchapter, since for the test problem we chose only thearticleelement.

Paragraph element

Element name:para.

Description: theparaelement represents a paragraph in the document. Usually this element is rendered as a block with padding before and after it.

Example:

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Ordered list element

Element name:orderedlist.

Description: ordered list is a list of items where each list item is marked with a number. Ordered lists can be used for describing a procedure which contains a number of steps. Ordered list is marked withorderedlistelement. Every list item in the list is represented by thelistitemelement. Content of each list item is wrapped by aparaelement. Besides, theorderedlistelement may also contain an optionalparaelement in the beginning to describe the content of the list.

Examples.

Element name:itemizedlist.

Description: theitemizedlistelement is used to add a simple unordered list to the document. Each list item is represented by a singlelistitemelement. All list item content is wrapped by a singleparaelement. Also a list may contain an optional para element at the beginning to describe contents of the list.

'command' element

Element name:command.

Description: thekeycapelement represents a text printed on a key on a keyboard.

Formatting: the content of this element should be formatted inline, the key caption is usually rendered with bold font.

Example:

PressF1to see a help window.

'emphasis' element

Element name:emphasis.

Description: theemphasiselement provides a method for indicating that certain text should be stressed in some way.

Formatting: the content of this element should be formatted inline, emphasized text is usually rendered with italic font.

Example:

Here is thethingwe would like to emphasize.

Optional inline elements task

Create a document with a fragment whereemphasisandkeycapelements are combined. Verify if you transformation supports them, extend it if needed.

Internal links

Element name:link.

Description: thelinkelement is used to organize links to other parts of the document. Links are organized using theidattribute. Each DocBook element can have anidattribute. To create a link to the particular element thelinkendattribute value of thelinkelement should contain the same value as theidattribute of the target element. The text within thelinkelement is the link text.

Formatting: the links text should be rendered with underlined font in green color.

Example:

Please clickhereto go to the Introduction chapter.

Links without a correct target set, likethis, should be highlighted but shouldn't take the reader to another place.

External links

Element name:ulink.

Description: theulinkelement is used to organize links to external resources, e.g. websites. Theurlattribute of theulinkelement should contain an URL of a site.

Formatting: the link text should be rendered with underlined font in green color.

Example:

Please clickhereto read more about external links in DocBook.

Optional link elements task

Bonus tasks

If you manage to complete the basic level transformation faster, here you can find a few bonus tasks. The complexity of bonus tasks is increased, so it makes sense to work on them in the provided order.

Tools

It doesn't matter which XSLT processor you use. However, we want to let you know that we are working in the Java environment and use Saxon XSLT processor in most of our projects.

It would be nice if you provide a simple Ant build which allows you to run a transformation.

Acceptance criteria

The acceptance procedure is very simple. We run you transformation to transform the source of this document, then review the resulting HTML document.

The transformation should be completed without errors. The resulting document should contain all the contents of the original document formatted accordingly.

Resources

This chapter contains links to various useful resources you may need to learn more about XML, XSLT, HTML and others. You can use any information sources you like (not listed here), but we decided to share with you some our favourite ones.
