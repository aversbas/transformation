XSLT Trainee Test Problem

Introduction

The document contains a test problem for applicants, a short introduction into DocBook and many links to resources where you can learn more about technologies you are supposed to work with: XML, XSLT, HTML and so on.

The test problem contains a few levels. We assume that a few days should be enough to learn all required technologies and develop a solution for the test problem. In case you complete this work faster then you can implement solutions for optional and bonus tasks. The more levels you complete, the better your chances are.

Block-level elements

Block-level elements in DocBook are elements like paragraphs, lists, and so forth. In the result HTML document they are usually represented with block-level elements. Next chapters briefly describe some of the block-level elements and provide ideas about how they can be rendered.

Paragraph element

Element name:para.

Description: theparaelement represents a paragraph in the document. Usually this element is rendered as a block with padding before and after it.

Example:

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Ordered list element

Element name:orderedlist.

Description: ordered list is a list of items where each list item is marked with a number. Ordered lists can be used for describing a procedure which contains a number of steps. Ordered list is marked withorderedlistelement. Every list item in the list is represented by thelistitemelement. Content of each list item is wrapped by aparaelement. Besides, theorderedlistelement may also contain an optionalparaelement in the beginning to describe the content of the list.

Examples.

The following list is a simple ordered list with four items. By default Arabic numbers are used to mark list items.

  1. Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

  2. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

  3. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur.

  4. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

The next list has a specific list intro element and 3 steps. List intro is a block level element (similar to paragraph) and should be appropriately indented (user should see that description corresponds to the list).

  3. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Unordered list element

Element name:itemizedlist.

Description: theitemizedlistelement is used to add a simple unordered list to the document. Each list item is represented by a singlelistitemelement. All list item content is wrapped by a singleparaelement. Also a list may contain an optional para element at the beginning to describe contents of the list.

Formatting: the list should be rendered as a simple bulleted list. When a list intro element is available it should be properly indented to let user see that it is a part of the list.

Examples:

Simple unordered list that contains four items:

  • Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Unordered list with a list intro and three list items:

Inline elements

Inline-level tags are elements like emphasis, hyperlinks, and so forth. They wrap a text within a block-level element. Usually the content of these elements is highlighted with a different font (different font family, font style, font weight etc).

'command' element

Element name:command.

Description: thekeycapelement represents a text printed on a key on a keyboard.

Formatting: the content of this element should be formatted inline, the key caption is usually rendered with bold font.

Example:

PressF1to see a help window.

'emphasis' element

Element name:emphasis.

Description: theemphasiselement provides a method for indicating that certain text should be stressed in some way.

Formatting: the content of this element should be formatted inline, emphasized text is usually rendered with italic font.

Example:

Here is thethingwe would like to emphasize.

Link elements

Link tags are used to create links which lead to another part of the document or to any external resource, for example a website.

Internal links

Element name:link.

Description: thelinkelement is used to organize links to other parts of the document. Links are organized using theidattribute. Each DocBook element can have anidattribute. To create a link to the particular element thelinkendattribute value of thelinkelement should contain the same value as theidattribute of the target element. The text within thelinkelement is the link text.

Formatting: the links text should be rendered with underlined font in green color.

Example:

Please clickhereto go to the Introduction chapter.

Links without a correct target set, likethis, should be highlighted but shouldn't take the reader to another place.

External links

Element name:ulink.

Description: theulinkelement is used to organize links to external resources, e.g. websites. Theurlattribute of theulinkelement should contain an URL of a site.

Formatting: the link text should be rendered with underlined font in green color.

Example:

Please clickhereto read more about external links in DocBook.

Implement table of contents

Extend the transformation to make it insert the table of contents (TOC) in the beginning of the document.

  1. TOC should have a list of all sections/titles encountered in the document

  2. TOC entries should have appropriate numbering and indentation regarding their position in the document hierarchy.

  3. TOC items should be links to the appropriate section/title, so when you click a title in the TOC, you are redirected to the corresponding section.

Implement multiple HTML page output

Extend the transformation to make it generate multiple HTML pages for the input document.

  1. Each document section should be presented as a separate HTML page.

  2. There should be anindex.htmlpage that contains the TOC.

  3. A click on a TOC entry should open the corresponding section in the same window

  4. If a section has child sections, then the corresponding page should contain mini-TOC names "Child Topics". Mini-TOC should have appropriate titles/links of/to child sections.

  5. Each page should have a navigation block at the top with links to previous, parent and next sections.

XML

XML (eXtensible Markup Language) is a set of rules for encoding documents electronically. It is defined in the XML 1.0 Specification produced by the W3C and several other related specifications; all are fee-free open standards. You can find a lot of information about XML at:

  • http://en.wikipedia.org/wiki/XML- Wikipedia page about XML, contains basic information about XML and a lot of links to useful XML-related resources.

XSLT

XSLT (XSL Transformations) is a declarative, XML-based language used for the transformation of XML documents into other XML (or non-XML) documents.

  • http://en.wikipedia.org/wiki/XSLT- Wikipedia page about XSLT, provides basic information about XSLT and a lot links to useful XSLT-related resources.

  • http://www.zvon.org/xxl/XSLTutorial/Output_rus/index.htm- one of the best XSLT tutorials/references with a lot of samples (in many languages including Russian).

  • http://xmlhack.ru/books/xslt- probably the best book about XSLT in Russian (partially available).

HTML

HTML, which stands for Hyper Text Markup Language, is the predominant markup language for web pages. It provides the means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists etc as well as for links, quotes, and other items.

  • http://en.wikipedia.org/wiki/HTML- Wikipedia page about HTML, provides basic information about HTML and a lot links to useful HTML-related resources.

  • http://www.w3schools.com/html/default.asp- excellent tutorials, online editor which allows you to quickly check how things work.

DocBook

DocBook is a semantic markup language for technical documentation. It was originally intended for writing technical documents related to computer hardware and software but it can be used for any other sort of documentation.

  • http://www.docbook.org/- the official DocBook home page.

  • http://en.wikipedia.org/wiki/DocBook- Wikipedia page about DocBook, provides basic information about DocBook and a lot links to useful DocBook-related resources.

  • http://www.docbook.org/tdg/en/html/docbook.html- DocBook: The Definitive Guide - very detailed book about DocBook format. Best place to search information about any element, DocBook document authoring and so on.
